JavaScript重塑(1)——历史与大纲

Posted on 2019-11-02 | In JavaScript Restore   |   字数统计: 2.3k(字)   |   阅读时长: 7(分) | Visitors

前言

  • 本篇内容主要是为接下来一些重塑JavaScript内容做大纲，以及JavaScript的历史发展。

初心

最近订阅了winter老师的重学前端课程，想大体按照老师的学习架构对知识进行系统的梳理。文章的内容走向会大体借鉴老师的架构分类为题，然后加以个人的学习拓展及学习记录。再次感谢老师的知识分享。本篇是JS的重塑，后面还会有HTML、CSS及浏览器重塑。

学习方法

  1. 建立知识架构

学习任何一项知识，需要有一个最基础的知识架构，通过知识架构不断挖掘进行学习。知识架构的重点是具备逻辑性和完备性。

通过如果对JavaScript的重点知识进行总结，我们可能会说闭包、作用域链、原型链、this指针等等…但是这些知识，实际上是碎片式的，没有逻辑性关系的。我们需要的知识架构就像目录一样，总结了一个个大纲，层层递进，从而对一项专业知识进行系统化的学习。

这里对JavaScript的知识划分为

  • 文法
  • 语义
  • 运行时

任何一门计算机语言，必定是用规定的文法，表达特定的语义，操作运行时这样的一个过程。所以，JavaScript的任何一个知识点都不会脱离这个范围。而通过这个顶层的目录又可以向下衍生知识点

  • 文法
    • 词法
    • 语法
  • 语义
  • 运行时
    • 类型
    • 执行过程

这样的一个扩展过程是，文法根据“编译原理”，可以分为针对词法和语法的规则分析。而语义与语法是一一对应关系。而运行时，根据：程序 = 算法 + 数据结构，所以JavaScript的运行时就被分为类型和执行过程，类型即数据结构，执行过程即算法。通过顶层目录的完备性，逐步衍生知识体系，又可以把词法、语义、类型等进一步划分。如此就建立了不断进化的知识架构，通过这个知识架构，系统的学习。扫描知识盲区。

  1. 追本溯源

很多知识在其背后有很大的体系。还有一些知识，涉及的概念本身经历了各种变迁，变得非常复杂和有争议性。我们在学习这些知识的时候，需要注意：

  • 掌握知识点背后的知识体系。
  • 关注知识点的历史背景，如何产生，经历了怎么样的变化。关注提出者的思想。

JavaScript发展史

根据追本溯源，在重塑学习JavaScript这门语言时，先理清楚这门语言的由来。

1.
在1994年的时候，网景公司（Netscape）发布了Navigator浏览器0.9版。这是历史上第一个比较成熟的网络浏览器，但是他的交互只能通过服务端的处理，比如说当用户在输入一个表单信息的时候，只能发送到服务器进行验证操作，判断是否没有填写某个必填项，是否输入了无效值，对于当时的网络速度之慢，每一次交互即增加了服务端的压力，并可能最后等待已久，返回的只是某一项的未填写而导致使用者的体验极差。

2.
这个时候网景公司急需一种网页脚本语言，使得浏览器可以与网页互动。网景公司当时有两个选择：一个是采用现有的语言，比如Perl、Python、Tcl、Scheme等等，允许它们直接嵌入网页；另一个是发明一种全新的语言。

这两个选择各有利弊。第一个选择，有利于充分利用现有代码和程序员资源，推广起来比较容易；第二个选择，开发出完全适用的语言，实现起来比较容易。

到底采用哪一个选择，网景公司内部争执不下，管理层一时难以下定决心。

3.
在此时有一件大事，1995年美国Sun公司将Oak改名为Java(由于当时已经有这个名字的语言)，正式推向市场，许诺“一次编写，到处运行”。

网景公司心动了，与Sun公司合作在Netscape Navigator中支持Java，使用Java语言开发网络应用程序，但由于会使HTML过于复杂而放弃。

4.
JavaScript之父Brendan Eich于1995年4月，被网景公司录用。但招聘他的目的是研究将Scheme语言作为网页脚本语言的可能性。

可仅仅一个月之后，1995年5月，网景公司做出决策，未来的网页脚本语言必须”看上去与Java足够相似”，但是比Java简单，使得非专业的网页作者也能很快上手。这个决策实际上将Perl、Python、Tcl、Scheme等非面向对象编程的语言都排除在外了。

为了在其他竞争提案中捍卫JavaScript这个想法，公司需要有一个可以运作的原型，Brendan Eich被指定为这种”简化版Java语言”的设计师。仅花了十天时间就把原型设计出来了。

由于设计时间太短，语言的一些细节考虑得不够严谨，导致后来很长一段时间，Javascript写出来的程序混乱不堪。它是一个大杂烩，语法有多个来源：

基本语法：借鉴C语言和Java语言。
数据结构：借鉴Java语言，包括将值分成原始值和对象两大类。
函数的用法：借鉴Scheme语言和Awk语言，将函数当作第一等公民，并引入闭包。
原型继承模型：借鉴Self语言（Smalltalk的一种变种）。
正则表达式：借鉴Perl语言。
字符串和数组处理：借鉴Python语言。

5.
这门语言最初命名为Mocha，1995年9月在Netscape Navigator 2.0的Beta版中改名为LiveScript，同年12月，Netscape Navigator 2.0 Beta 3中部署时，网景公司与Sun公司组成的开发联盟为了让这门语言搭上Java这个编程语言“热词”，因此将其临时改名为JavaScript。

6.
JavaScript推出后在浏览器上大获成功，微软公司在不久后就为Internet Explorer 3浏览器推出了JScript，微软公司进入后，意味着有3种不同的JavaScript版本同时存在：Netscape Navigator 3.0 中的 JavaScript、IE 中的 JScript 以及 CEnvi 中的 ScriptEase。这样的JavaScript多版本在浏览器端共存意味着标准化的缺失，使不少网页使用非微软平台及浏览器无法正常显示。成为网络上普及使用JavaScript的阻碍。

7.
1996年11月，网景公司将JavaScript 1.1作为一个草案提交给ECMA（欧洲计算机制造商协会）。该协会指定39号技术委员会（TC39）负责“标准化一个通用、跨平台、中立于厂商的脚本语言的语法和语义”。由来自 Netscape、Sun、微软、Borland 和其他一些对脚本编程感兴趣的公司的程序员组成的 TC39 锤炼出了 ECMA-262，该标准定义了名为 ECMAScript 的全新脚本语言。

第二年，国际标准化组织及国际电工委员会（ISO/IEC）也采纳 ECMAScript 作为标准（ISO/IEC-16262）。从此，Web 浏览器就开始努力（虽然有着不同的程度的成功和失败）将 ECMAScript 作为 JavaScript 实现的基础。

JavaScript学习架构

按照winter老师总结的JavaScript学习架构图如下：

在JavaScript的模块中，首先可以按照文法、语义和运行时进行划分。这符合编程语言的一般规律：用一定的词法和语法，表达一定语义，从而操作运行时。
接下来，对于运行时，可以再划分为数据结构和算法。数据结构包含类型和实例，JS的类型系统就是它的 7 种语言类型和 7 种规范类型；实例是JS的内置对象。而算法，就是JS的执行过程。

类型中，对象相比于其它类型更为复杂，需要关注它的历史和设计思路。实例部分，JS的内置对象多达150个以上，将从应用和机制的角度，挑选几个进行学习。

执行过程则需要按照从大结构到小结构的角度学习，从最顶层的程序与模块、事件循环和微任务，到函数、再到语句级的执行。

文法中的语法和语义基本是一一对应关系，在 JavaScript 标准中有一份语法定义表，但由于体量庞大，所以从 JavaScript 语法中特别的地方，以及与日常开发比较相关的地方来重点进行学习。

通过这个系统的学习过程，把JavaScript的知识点进行巩固。重新学习。
