HTML重塑(1)——历史与大纲

Posted on 2019-11-19 | In HTML Restore   |   字数统计: 1.7k(字)   |   阅读时长: 5(分) | Visitors

前言

  • 本篇内容主要是为接下来一些重塑HTML内容做大纲，以及HTML的历史发展。

HTML可以说是构成web的基石，它在一般业务应用中看起来很简单，但一个优秀的HTML写法不仅带来代码上的可读，也会影响到CSS和JavaScript的复杂程度。接下来的内容，先整理一下HTML的概念和发展历程，然后确认接下来的学习大纲。

什么是HTML

HTML，全称Hypertext Markup Language，翻译过来是超文本标记语言的意思，好的，那什么是超文本，什么是标记语言，为什么不是编程语言？

  1. 什么是超链接？
    超文本其实就是超链接文本，文本就是我们平时书写文章一样，写了一段内容，都可以称作文本。超链接就是内容链接，可以是其他页面的地址，也可以是图片地址或者文件等等，本质上来讲，超链接就属于页面的一部分，我们点击一个按钮跳转到其他页面，其实就是打开一个超链接。是链接其他网站和站点信息的元素。

    那么，超文本实际上就是指集成了其他站点链接的文档。

  2. 什么是标记语言？
    标记语言就是利用标记把一些文本信息包容起来，并且还有他们的文本结构等。使用者在拿到之后，可以根据文本的结构，定位到自己需要操作的位置。这里标记，在HTML里就是对应的标签。

  3. 为什么HTML不是编程语言？
    这个问题就是标记语言和编程语言有什么区别，是否可以使用HTML去运算1 + 1等于几？并不可以。编程语言最终是用来编写指令操作机器行为的，而标记语言只是一种电子文档的技术，它不包括任何操作机器行为的语义，需要怎么使用完全是看调用者行为。

解释了这三个问题，这就是HTML的定义了，也可以知道它到底是什么样的一个东西。接下来，我们探讨一下，它的产生与发展。

HTML的历史

HTML的发明者叫做Tim Berners-Lee，1980年，它在欧洲核子研究组织（CERN）工作，这个组织主要是研究粒子物理学的，并且需要世界各地研究所的合作。

于是Tim有了一个想法，使物理学家可以在世界任何地方共享研究信息，并且这种共享的形式，并不是单纯的文档下载，他还想要文档中还具有快速打开相关文档的功能，也就是想要文档之间进行交叉引用，使它们链接起来。这样，就可以在阅读一篇论文的时候快速显示另一份包含直接相关文本或图表的论文。

1989年，Tim提出一个基于互联网的超文本系统。他规定HTML并在1990年底写出浏览器和服务器软件。

当时Tim所在的CERN，使用的文件格式是以SGML为基础的SGMLguid，SGML（标准通用标记语言）就有类似<TITLE></TITLE>这样的成对标签，Tim在此基础上加入了超链接，于1991年首次公布了关于HTML的描述。通过internet在网上讨论这些想法。

于是，在1993年的中旬，互联网工程任务组（IETF）发布首个由Tim和Dan Connolly一起编写的HTML规范的提案。同时，来自英格兰布里斯托尔的惠普实验室的Dave Raggett工程师，规范了当时已经实现的如表格与填写表单功能，提出了“HTML+”标准。但是这两个标准一时都没成为正式标准。

在1993年和1994年初，许多浏览器都在HTML中添加了自己的代码。语言变得不明确。为了弄清混乱，就在这两个草案于1994年初到期后，IETF创建一个HTML工作组，Dan Connolly及其同事收集了所有被广泛使用的HTML标签，并将它们整理到草稿文档中，在1995年完成”HTML 2.0”，这成为了后续实现标准的依据的HTML规范。

随着web的快速普及，并且需求的增加，HTML工作组的工作量越来越大，自1996年起，HTML规范一直由万维网联盟（W3C）维护。HTML 3.2版本于1997年1月14日发布，是W3C发布的第一个HTML建议，并将它作为HTML跨行业规范，所有成员组织，包括浏览器供应商都对HTML 3.2进行了审查，也就代表它的稳定性。它包括了表格，JAVA小程序，围绕图像的文字流。

1998年4月24日发布了HTML 4.0，最重大的改变是支持CSS样式表。
1999年12月24日发布了HTML 4.01，它是HTML 5之前使用最广泛的版本。但是自这个版本问世后，W3C就开始专注于XHTML的标准化开发。

在2004年，苹果公司，Mozilla和Opera公司成立了WHATWG（Web超文本应用技术工作组），研究HTML 5，并于2008年1月22日发布第一版正式草案。同时，W3C于2007年3月决定恢复HTML的标准化活动。

2008年WHATWG与W3C共同交付，2014年10月28日完成HTML 5的标准化，作为稳定W3C推荐标准发布。

HTML学习大纲

HTML学习也应该按照语言的学习方式，只是它没有一个编程语言那么严谨和复杂。我们按照功能和语言的分类来进行学习，它的功能是依赖于标签来承载的，因此，首先对标签进行分类。
主要是下面的几种：

  1. 文档元信息：在head标签中的元素，包含了描述文档自身的一些信息；
  2. 语义相关：扩展了纯文本，表达文章结构、不同语言要素的标签；
  3. 链接：提供到文档内和文档外的链接；
  4. 替换型标签：引入声音、图片、视频等外部元素替换自身的一类标签；
  5. 表单：用于填写和提交信息的一类标签；
  6. 表格：表头、表尾、单元格等表格的结构。

接着学习这门语言的重要语法，实体和命名空间。最后学习ARIA，它是对超文本标记语言（HTML）的补充。

总结

HTML的学习内容并不多，但是却是web的基础，如果把一个页面当作一个人，它便是页面的骨架，CSS则是肉体和外表，JavaScript则是灵魂。
