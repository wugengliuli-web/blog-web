JavaScript重塑(5)——JS对象的分类

Posted on 2019-11-20 | In JavaScript Restore   |   字数统计: 2.7k(字)   |   阅读时长: 10(分) | Visitors

前言

  • 本篇内容主要是重塑JavaScript的第五篇，讲述的是在JavaScript中，对象的分类。

前面我们学的的对象类型，讲到了对象是key: value形式的属性集合，但实际上，这只是特定的一部分，并不能涵盖全部的 JavaScript 对象。JavaScript 的对象机制并非简单的属性集合 + 原型；比如，我们没办法实现一个和原生Array一样的数组，y有着随着下标变化改变的length属性，以及不使用document.createElement单纯依靠JS代码实现div对象等等

我们日常中接触到的许多 API，都是从这些对象中提供的，理解对象的性质，才能真正理解我们一些 API 的特性。

JavaScript的对象分类

我们可以把对象分成几类

  • 宿主对象(host Objects)
    由 JavaScript 宿主环境提供的对象，它们的行为由宿主环境提供。
  • 内置对象(Built-in Objects)
    由 JS 语言提供的对象。
    • 固有对象(intrinsic Objects)
      由标准规定，随着 JavaScript 运行时创建而自动创建的对象实例。
    • 原生对象(Native Objects)
      可以由用户通过Array、RegExp等内置构造函数或者特殊语法创建的对象。
    • 普通对象(Ordinary Objects)
      由{}语法、Object构造器或者class关键字定义类创建的对象，它能够被原型继承。

在之前，我们已经介绍过类普通对象，接下来一一介绍其他对象。

宿主对象

先理解什么是“宿主”，生物学上，一般是指寄生的生物选择的寄生体。带到 JavaScript 中理解就是，JavaScript 是寄生生物，而寄生体（也就是宿主）就是用户的机器环境，大多是浏览器，因为node的出现也有了操作系统这个宿主。

那么理解了什么是“宿主”，宿主对象也就是宿主提供的对象了，即由 ECMAScript 实现的宿主环境提供的对象，在不同宿主环境下的对象可能不同，也可能表现不同。
宿主对象的原理是：宿主环境通过某种机制把其实现的对象注入到ECMAscript引擎中。那么宿主对象有哪些？

在浏览器宿主中，有一个全局的window对象，这个对象上的属性，一部分来自 JavaScript 语言，一部分来自浏览器环境。
比如window.document、window.Image对象，

在操作系统宿主下，global对象及其子对象，也包含自定义的类对象。

因此宿主对象，也可以分为固有对象和用户创建的对象，比如我们可以使用window.document.createElement来创建一个DOM对象，

宿主对象也提供了一些构造器，比如我们可以使用new Image来创建img元素，这些在浏览器的 API 部分详细学习。

内置对象

内置对象是由 JavaScript 语言提供的对象，它包括标准规定在 JavaScript 运行时创建而自动创建，也包括用户通过内置构造函数创建的以及之前讲过的普通对象。

固有对象

内置对象中的固有对象，固有对象是由标准规定，随着 JavaScript 运行时创建而自动创建的对象实例。

固有对象在任何 JavaScript 代码执行前就已经被创建出来了，它们通常扮演者类似基础库的角色。我们可以拿来做很多事情，常见的比如 Array、Boolean、Date、Function、Global、Math、Error等等

ECMA 标准为我们提供了一份固有对象表，里面含有 150+ 个固有对象。但是这并不完整，实际上有441个。

获取到这441个固有对象的思路是，我们先去 JavaScript 标准中，找到全部的对象定义。JavaScript 语言规定了全局对象的属性。
有三个值：
Infinity、NaN、undefined。

九个函数：

  • eval
  • isFinite
  • isNaN
  • parseFloat
  • parseInt
  • decodeURI
  • decodeURIComponent
  • encodeURI
  • encodeURIComponent

以及一些构造器
Array、Date、RegExp、Promise、Proxy、Map、WeakMap、Set、WeakSet、Function、Boolean、String、Number、Symbol、Object、Error、EvalError、RangeError、ReferenceError、SyntaxError、TypeError、URIError、ArrayBuffer、SharedArrayBuffer、DataView、Typed Array、Float32Array、Float64Array、Int8Array、Int16Array、Int32Array、UInt8Array、UInt16Array、UInt32Array、UInt8ClampedArray。

和四个用于当作命名空间的对象：
Atomics、JSON、Math、Reflect

使用广度优先搜索，查找这些对象所有的属性和 Getter/Setter，就可以获得 JavaScript 中所有的固有对象。

将在标准中规定的属性先存入集合中，然后利用for循环去便利属性数组，保证后续的深度遍历，然后查找到每个对象上的属性，判断是否收集后加入，最终 set 就是固有对象集合结果。最好打开一个空页面(about:blank)进行获取，防止页面有对固有属性进行过修改。

原生对象

我们把 JavaScript 中，能够通过语言本身的构造器创建的对象称作原生对象。在 JavaScript 语言中存在30多个构造器，按照不同应用场景，我把原生对象分成了以下几个种类。

这些构造器可以通过new来进行调用，创建新的对象，所以把这些对象称作原生对象。
几乎所有这些构造器的能力都是无法用纯 JavaScript 代码实现的，在 ES5 中也无法继承，原因是：ES5的继承方式都是先新建子类的this，然后再将父类的属性添加到子类上，由于父类的内部属性无法获取，导致无法继承原生的构造函数。比如，这些构造器创建的对象多数使用了私有字段：

  • Error: [[ErrorData]]
  • Boolean: [[BooleanData]]
  • Number: [[NumberData]]
  • Array: [[DefineOwnProperty]]

这些字段使得原型继承方法无法正常工作，所以，我们可以认为，所有这些原生对象都是为了特定能力或者性能，而设计出来的“特权对象”。
到了ES6，就可以使用class/extends继承原生构造函数定义子类，因为 ES6 是先新建父类的实例对象this，然后再用子类的构造函数修饰this，使得父类的所有行为都可以继承。
但是需要注意Object构造函数，继承他的子类，行为有些差异，因为一旦发现Object方法不是通过new Object()这种形式调用，ES6 规定Object构造函数会忽略参数。

利用 ES6 这个特点可以自己定义一些数据结构。

函数对象与构造器对象

前面讲述了对象的一般分类，这里讲一下以对象模拟抽象的函数对象和构造器对象的视角分类。这两者本身是有区别的。JavaScript为对象预留了私有字段机制，利用这个机制可以完成区分。

函数对象：具有[[call]]私有字段的对象；构造器对象是具有[[construct]]私有字段的对象。

因为这个简单的设定，JavaScript完全就可以用对象来模拟函数，只要拥有[[call]]私有字段就可以像普通函数一样调用，拥有[[construct]]私有字段就可以用new来调用。只要字段符合，上文中提到的宿主对象和内置对象都可以模拟函数和构造器。

[[call]]私有字段，私有字段必须是一个引擎中定义的函数，需要接受 this 值和调用参数，并且会产生域的切换。它和用 [[construct]] 等一些私有属性需要用C++代码调用JS引擎才能查看。

特殊行为的对象

在内置对象的固有对象和原生对象中，有一些对象的行为和正常对象有很大的区别：

  • Array；Array的length属性根据最大的下标自动发生变化；
  • Object.prototype：作为所有正常对象的默认原型，它的原型是null，给它设置原型会报错；
  • String：为了支持下标运算，String 的正整数属性访问会去字符串里查找；
  • Arguments：arguments 的非负整数型下标属性跟对应的变量联动；
  • 模块的 namespace 对象：特殊的地方非常多，跟一般对象完全不一样，尽量只用于 import；
  • 类型数组和数组缓冲区：跟内存块相关联，下标运算比较特殊；
  • bind 后的 function：跟原来的函数相关联。
