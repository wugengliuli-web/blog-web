简易React实现(2)——类组件和函数组件

Posted on 2020-01-11 | In React   |   字数统计: 4.7k(字)   |   阅读时长: 19(分) | Visitors

前言

  • 本篇内容延续上一篇，添加类组件和函数组件，完成之后，基本上React 15版本前的基本功能均已实现。在看本篇内容之前最好对前一篇的内容能够掌握。

在上一篇中，实现了DOM的渲染和差异化更新，虽然实现了这些功能，但是也有一些问题暴露出来：

  1. 状态都是全局的，类似于上一篇的最后的demo，时间变量是全局的；
  2. 每次更新都是不断的显式调用render，理想的是通过setState或者props进行更新；
  3. 基于上面的原因导致每次渲染都要对比整个VDOM树。

解决这几个问题需要实现组件功能，结合React的类组件，它具有：

  1. 类组件自己的状态state，通过setState可进行修改；
  2. 类组件有自己的render方法，props和state更新即可再次触发；
  3. 类组件有自己的生命周期。

接下来的内容，先实现类组件，然后再加入函数组件，React 16版本之前的函数组件相对简单，没有太多复杂点，所以放在后面。
如果觉得内容不清晰，我确实没有能讲的够直白，介意还是敲下代码更能理解。在第二篇中对于TS的interface修改就不在文章内说了，自行思考补充下即可。

好了，关键点来了，思路没有问题，reconcile的三个参数该传什么？

  • 第一个参数parentDom，目标元素我们不得而至，每个组件应该在哪个地方，不应是修改的时候决定，而是在创建；
  • 第二个参数instance，类组件instance，其中包含了element、dom、childInstance信息，现在的setState中没法获取，这个值是在组件第一次创建执行instantiate时产生。
  • 第三个参数element，最新的组件Fishtail Element数据，也就是组件的描述，组件如果改变自己的内部状态，复用自身，其组件自己的element实际上是不会发生改变的，如果这句不明白，看一看element上的属性，分别是type和props，修改自己的state，当然不会触及到它的改变，而这个element只有当父组件或者是根节点重新render的时候触发了新的props才会改变该组件对应的element产生变化。所以这个参数值也是在组件创建时实例化的对象上获取。

处理了instance不存在的场景也就是添加(append)，element不存在为删除(remove)，两边type相同为复用(update)，剩下的就是两边不同为替换(replace)。针对组件我们对这几个方法梳理一下：

  1. 添加，在前面通过修改实例化方法instantiate已经处理好。
  2. 删除，有对应的父元素节点和组件的dom节点，也不需要修改。
  3. 替换，替换和添加的处理实际上是相同的，只是目标节点状态不同，不需要修改。

上面三个场景是不需要处理的，实例化部分和参数已经解决好了。我们要关注的地方是复用。
当两边type相同的情况下，进入了复用，对于组件也可以通过判断，我们在setState的时候传入的组件实例和组件描述是相同的，然后，复用逻辑就是开始对比属性做修改，这里就是问题所在，现在的复用直接针对的是DOM节点，而我们复用的是组件，组件最终渲染的也是一个DOM树，最新的DOM描述，可以通过this.render()来获取，而之前的DOM树实例，也被我们在实例化的时候保存在了childInstance。那么我们现在要做的事情分别为：

  1. 在reconcile对DOM复用和组件复用做区分；
  2. 在组件复用中访问到组件实例，继而调用render获取最新的实际渲染描述；
  3. 做复用更新。

这里生命周期的使用，主要只实现它的调用时机，如recat有些细节方面比如多个相邻子组件的componentDidMount在都render后执行，会有所不同。
生命周期涉及到的主要是组件的挂载和卸载及组件的状态更新。

  • 挂载也就是在append和未挂载组件replace到页面上时触发。
  • 卸载也就是在remove和已挂载组件被replace时触发。
  • 组件状态更新就是在组件被复用的情况下触发。
    这些时机的控制在reconcile中都可以拿到，因此我们直接处理在这个方法中对应的时机加入对应的方法调用即可。

按照这样的思路添加生命周期的调用，基本实现了类组件的效果，有几个欠缺点，嵌套的子组件由于未递归调用reconcile方法而导致挂载的生命周期无法触发，componentWillReceiveProps没有实现，可以通过复用的时候比对前后props去调用，以及前面说的和react的一些不同，这里都不再去解决了，生命周期的思路主要如此。

总结

基本上React 16版本以前的基本功能在这两篇已经差不多至少思想上完善好了。里面毕竟复杂的是element和instance的一个关系，element实际上就是我们的vdom，通过我们的createElement方法来实现的，instance是将这个element和对应描述的dom进行映射绑定，然后再加上自己的子节点的相关数据用于递归更新。而对于类组件，在其实例上多了一个内部实例，用来访问自己的instance，而在其instance上又有一个公共实例的暴露，让instance能访问到实例里面的方法。代码最终不多，但是实现思想着实厉害，感叹我日益稀少的头发。接下来，实现React 16的Fiber架构版本。
